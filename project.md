# ReForge — Product Specification

## Product Vision

ReForge is an AI-powered software archaeology workspace that helps users discover, understand, restore, and evolve software projects that deserve another chapter.

It is not a chatbot.

It is an interactive workspace where software is explored, reconstructed, and gradually brought back to life through collaboration between humans and AI agents.

---

# Target Users

### Primary

* Open-source maintainers
* Software engineers
* Students
* Researchers
* Digital preservation enthusiasts

### Secondary

* Universities
* Museums
* Historical archives
* Organizations maintaining legacy software

---

# User Problem

Developers constantly encounter abandoned software that is:

* impossible to build
* poorly documented
* dependent on obsolete frameworks
* missing contributors
* difficult to understand

Current AI tools help developers write new software.

Almost none help developers understand software that already exists.

---

# User Goal

The user wants to answer one question:

> **"Can this software live again?"**

---

# User Journey

## Step 1

Paste a GitHub repository.

or

Upload a project.

---

## Step 2

ReForge begins excavation.

Instead of immediately generating code, agents begin investigating.

Progress appears live.

Examples:

✔ Discovering project structure

✔ Recovering dependencies

✔ Reading commit history

✔ Understanding architecture

✔ Detecting build system

---

## Step 3

The user receives an Archaeology Report.

Not code.

Knowledge.

Example sections:

* Project Summary
* Purpose
* Technology Stack
* Repository Health
* Build Status
* Missing Components
* Historical Timeline
* Restoration Potential
* Evolution Opportunities

---

## Step 4

The user decides.

Possible actions:

* Restore Build
* Modernize
* Explore Architecture
* Analyze Dependencies
* Continue Restoration

The human remains in control.

---

# Core Product Principles

## Understand Before Restore

No code should be modified until the software has been understood.

---

## Human First

The AI recommends.

The user decides.

---

## Explain Everything

Every recommendation should explain:

* why
* how
* impact

Nothing should be a black box.

---

## Preserve Identity

ReForge should avoid changing what makes a project unique.

Technology may evolve.

Identity should remain.

---

# Core AI Agents

## Repository Explorer

Discovers project structure.

---

## Historian

Reconstructs purpose and development history.

---

## Architect

Explains architecture and design.

---

## Builder

Attempts compilation and execution.

---

## Restorer

Repairs build failures.

---

## Evolution Planner

Suggests future improvements.

---

# First Wow Moment

Within two minutes of importing a repository, the user should feel:

> "I understand this project better than I ever could by reading the README."

---

# Long-Term Vision

Eventually, ReForge becomes the world's largest software archaeology platform.

Not every project deserves restoration.

But every project deserves understanding.

ReForge helps identify the software worth preserving and gives it the opportunity to continue evolving.

---

# Product Promise

ReForge does not simply modernize software.

It helps software continue its story.
