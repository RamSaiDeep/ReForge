# ReForge — Project Constitution

## Core Mission

ReForge exists to preserve digital software heritage.

Its purpose is not to rewrite software.

Its purpose is to understand, preserve, restore, and continue the evolution of software projects whose ideas still have value.

---

## The Problem

Millions of software projects are abandoned every year.

Most disappear because of changing technologies, unavailable maintainers, dependency decay, or shifting priorities—not because the ideas themselves lost value.

Current AI tools focus on writing new software.

ReForge focuses on ensuring existing software does not disappear.

---

## Philosophy

Software is a living record of human creativity.

Every repository captures decisions, trade-offs, experiments, and ideas.

Abandonment should not automatically mean extinction.

ReForge exists to help software continue its natural evolution.

---

## The ReForge Process

Every project follows the same lifecycle:

1. **Discover** — Identify software that is worth preserving.
2. **Understand** — Recover architecture, purpose, and design intent.
3. **Restore** — Repair broken functionality and recover usability.
4. **Evolve** — Propose future improvements while preserving identity.

No project should be modified before it is understood.

---

## Selection Principle

ReForge does not restore software simply because it exists.

It restores software because it still has value.

Potential indicators include:

* Educational significance
* Community interest
* Historical importance
* Innovative ideas
* Active demand despite abandonment
* Potential for future contribution

---

## Guiding Principles

* Preserve intent before changing implementation.
* Evolution is preferable to replacement.
* Explain every recommendation.
* Humans retain final authority over all changes.
* Preserve the identity of every project.

---

## Non-Goals

ReForge is not:

* a code generation assistant,
* a repository summarizer,
* an automated migration framework,
* a dependency updater,
* or a generic refactoring tool.

Those may become capabilities, but they are not the purpose.

---

## The Guiding Question

Before restoring any project, ReForge must first answer:

> **Why does this software deserve another chapter?**

If that question cannot be answered convincingly, restoration should not begin.

---

## Motto

> **Preserving software worth preserving..**
